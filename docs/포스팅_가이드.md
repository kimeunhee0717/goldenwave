## 1. 시작하기 전 준비

### 1.1 서버 실행하기

포스팅을 하려면 **2개의 서버**를 켜야 합니다.

#### 터미널 1: API 서버 실행
```bash
cd C:\NewProject\bujatime\goldenwave
node local-api.cjs
```
성공 메시지:
```
로컬 API 서버 시작: http://localhost:18790
```

#### 터미널 2: 개발 서버 실행
```bash
cd C:\NewProject\bujatime\goldenwave
npm run dev
```
성공 메시지:
```
VITE vX.X.X ready in XXX ms
➜  Local:   http://localhost:5173/
```

> **주의:** 포트 번호가 다를 수 있습니다 (5173, 3000, 3001 등). 터미널에 표시된 포트를 확인하세요.

### 1.2 Admin 페이지 접속

1. 브라우저에서 `http://localhost:5173/admin` 접속
2. 비밀번호 입력 (설정된 비밀번호)
3. "새 포스트 작성" 버튼 클릭 또는 `/admin/new` 직접 접속

## 2. 포스트 작성하기

### 2.1 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│ [← 포스트 목록]  새 포스트 작성    [미리보기] [저장] [발행] [배포] │
├─────────────────────┬───────────────────────────────────────┤
│   포스트 정보       │           마크다운 편집기              │
│                     │                                       │
│ [제목 입력]         │  [이미지 추가]                  0자   │
│ [슬러그 입력]       │ ┌───────────────────────────────────┐ │
│ [카테고리 선택]     │ │                                   │ │
│ [요약 입력]         │ │   본문을 입력하세요...            │ │
│ [커버 이미지 URL]   │ │                                   │ │
│ [태그 추가]         │ │   이미지를 드래그해서 놓거나      │ │
│ □ 추천 글 설정      │ │   위의 '이미지 추가' 클릭         │ │
│                     │ │                                   │ │
│                     │ └───────────────────────────────────┘ │
└─────────────────────┴───────────────────────────────────────┘
```

### 2.2 필수 입력 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **제목** | 포스트 제목 (필수) | "스마트폰 용량 부족 해결법" |
| **슬러그** | URL에 사용될 영문 ID (필수) | `smartphone-storage-fix` |
| **카테고리** | 포스트 분류 (필수) | 디지털 생활, 머니플러스 등 |
| **본문** | 마크다운 형식의 본문 (필수) | ... |

### 2.3 선택 입력 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **요약** | 검색 결과/목록에 표시될 요약 | "용량 부족 알림이 자꾸 뜰 때..." |
| **커버 이미지** | 대표 이미지 URL | `https://images.unsplash.com/...` |
| **태그** | 관련 키워드들 | `스마트폰`, `저장공간`, `팁` |
| **추천 글** | 메인에 노출할 추천 글 여부 | 체크/해제 |

### 2.4 슬러그 작성 규칙

- **영문 소문자**만 사용
- **숫자** 사용 가능
- **하이픈(-)** 으로 단어 연결
- 공백, 한글, 특수문자 사용 불가

```
✅ 좋은 예:
smartphone-storage-fix
ai-beginner-guide-2026
money-saving-tips-01

❌ 나쁜 예:
스마트폰용량 (한글 사용)
Smartphone Storage (대문자, 공백)
ai_guide (언더스코어)
```

### 2.5 카테고리 선택

| 카테고리 | 설명 | 적합한 콘텐츠 |
|----------|------|---------------|
| **디지털 생활** | AI, 디지털 기기, 앱 활용법 | ChatGPT 사용법, 스마트폰 팁 |
| **머니플러스** | 재테크, 투자, 부업 | 적금 비교, 주식 기초, N잡 |
| **부자마인드** | 마인드셋, 성공 습관 | 부자들의 습관, 시간관리 |
| **브리핑** | 뉴스, 트렌드 정리 | 오늘의 경제 브리핑 |
| **편집중** | 미완성 초안 보관 | 작성 중인 글 |

## 3. 이미지 추가하기

### 3.1 이미지 추가 방법 (2가지)

#### 방법 1: 드래그 앤 드롭 (추천)

1. 탐색기에서 이미지 파일 선택
2. 편집기 영역으로 **드래그**
3. 노란 안내 화면이 나타나면 **드롭**
4. 자동으로 업로드 & 마크다운 삽입

```
┌─────────────────────────────────────┐
│                                     │
│     🖼️ 이미지를 여기에 놓으세요     │
│                                     │
└─────────────────────────────────────┘
```

#### 방법 2: 파일 선택 버튼

1. 편집기 상단의 **[이미지 추가]** 버튼 클릭
2. 파일 선택 대화상자에서 이미지 선택
3. 자동으로 업로드 & 마크다운 삽입

### 3.2 이미지 업로드 과정

```
1. 이미지 선택/드롭
      ↓
2. Base64로 인코딩
      ↓
3. API 서버로 전송 (POST /api/upload-image)
      ↓
4. public/images/posts/ 폴더에 저장
      ↓
5. 저장된 URL 반환
      ↓
6. 커서 위치에 마크다운 삽입
   ![이미지명](/images/posts/1708300000-abc123.jpg)
```

### 3.3 이미지 제한사항

- **허용 형식:** JPG, PNG, GIF, WebP 등 이미지 파일
- **최대 크기:** 10MB
- **저장 위치:** `public/images/posts/`
- **파일명:** 자동 생성 (타임스탬프-랜덤문자열)

### 3.4 이미지가 저장되는 위치

```
goldenwave/
├── public/
│   └── images/
│       └── posts/
│           ├── 1708300000000-abc123.jpg  ← 새로 업로드된 이미지
│           ├── 1708300001234-def456.png
│           └── ...
```

빌드 시 `dist/images/posts/`로 복사됨

### 3.5 삽입되는 마크다운 형식

```markdown
![파일명](/images/posts/1708300000000-abc123.jpg)
```

- `파일명`: 원본 파일명에서 확장자 제거
- URL: 항상 `/images/posts/`로 시작

### 3.6 이미지 위치 조정

이미지가 삽입된 후 **커서 위치를 변경**하면 다음 이미지는 그 위치에 삽입됩니다.

```markdown
## 첫 번째 섹션

여기에 첫 번째 이미지를 넣고 싶다면,
이 줄 끝에 커서를 두고 이미지를 추가하세요.
[커서] ← 여기에 이미지가 들어감

## 두 번째 섹션

이 아래에 두 번째 이미지를 넣고 싶다면,
[커서] ← 여기에 커서를 두세요
```

## 4. 저장/발행/배포 단계

### 4.1 3단계 워크플로우

```
[저장] ──→ [발행] ──→ [배포]
  │          │          │
  ↓          ↓          ↓
파일 저장   빌드     Git Push
(로컬)    (dist)   (Vercel)
```

### 4.2 Step 1: 저장

**버튼:** 회색 "저장" 버튼

**하는 일:**
- `src/data/posts.json`에 포스트 정보 추가
- `content/{category}/{slug}.md` 파일 생성

**저장 후:**
- 버튼이 초록색 "저장됨"으로 변경
- "발행" 버튼 활성화

**언제 사용?**
- 글을 작성하면서 수시로 저장
- 아직 공개하지 않을 초안 저장

### 4.3 Step 2: 발행

**버튼:** 파란색 "발행" 버튼

**하는 일:**
- `npm run build` 실행
- 모든 페이지의 HTML을 dist 폴더에 생성 (프리렌더링)
- 새 포스트의 `/blog/{slug}/index.html` 생성

**발행 후:**
- 버튼이 파란색 "발행됨"으로 변경
- "배포" 버튼 활성화

**언제 사용?**
- 글 작성 완료 후
- 실제로 사이트에 공개하기 직전

**소요 시간:** 약 30초 ~ 1분

### 4.4 Step 3: 배포

**버튼:** 노란색 "배포" 버튼

**하는 일:**
- `git add .` - 모든 변경사항 스테이징
- `git commit -m "Add blog post: {제목}"` - 커밋
- `git push` - GitHub에 푸시
- Vercel이 자동으로 감지해서 배포

**배포 후:**
- 버튼이 초록색 "배포완료"로 변경
- 1~2분 후 실제 사이트에서 확인 가능

**언제 사용?**
- 최종 검토 완료 후
- 실제로 인터넷에 공개할 때만!

### 4.5 단계별 상태 표시

| 상태 | 저장 버튼 | 발행 버튼 | 배포 버튼 |
|------|-----------|-----------|-----------|
| 초기 | 회색 [저장] | 비활성화 | 비활성화 |
| 저장 중 | [로딩] 저장중 | 비활성화 | 비활성화 |
| 저장 완료 | 초록 [저장됨] | 파란 [발행] | 비활성화 |
| 발행 중 | 초록 [저장됨] | [로딩] 발행중 | 비활성화 |
| 발행 완료 | 초록 [저장됨] | 파란 [발행됨] | 노란 [배포] |
| 배포 중 | 초록 [저장됨] | 파란 [발행됨] | [로딩] 배포중 |
| 배포 완료 | 초록 [저장됨] | 파란 [발행됨] | 초록 [배포완료] |

### 4.6 수정 후 재배포

글 수정 시 동일한 단계를 반복합니다:

1. 내용 수정
2. **저장** (파일 업데이트)
3. **발행** (dist 재생성)
4. **배포** (변경사항 푸시)

## 6. 문제 해결

### 6.1 "로컬 API 서버가 꺼져 있습니다" 오류

**원인:** API 서버가 실행되지 않음

**해결:**
```bash
cd C:\NewProject\bujatime\goldenwave
node local-api.cjs
```

### 6.2 저장 버튼이 작동하지 않음

**확인사항:**
1. 제목 입력했는지?
2. 슬러그 입력했는지? (영문 소문자만)
3. 본문 입력했는지?
4. API 서버 실행 중인지?

### 6.3 이미지 업로드 실패

**확인사항:**
1. 파일이 이미지인지? (JPG, PNG 등)
2. 파일 크기가 10MB 이하인지?
3. API 서버 실행 중인지?

### 6.4 발행(빌드) 실패

**확인사항:**
1. 터미널에서 오류 메시지 확인
2. `npm run build` 직접 실행해서 오류 확인

**일반적인 원인:**
- TypeScript 타입 오류
- import 경로 오류

### 6.5 배포(푸시) 실패

**확인사항:**
1. Git 로그인 상태 확인
2. 원격 저장소 접근 권한 확인

**해결:**
```bash
git status
git push origin main
```

### 6.6 미리보기가 깨져 보임

**원인:** 마크다운 문법 오류

**확인사항:**
- 이미지 경로가 올바른지 (`/images/posts/...`)
- 코드 블록이 닫혔는지 (``` 짝이 맞는지)
- 표 문법이 올바른지
