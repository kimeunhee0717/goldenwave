# 블로그 개발 옵션 비교

> 옵션 A (Next.js) vs 옵션 B (Vite) 한눈에 비교
>
> 상세 내용은 각 옵션 문서 참조:
> - [옵션 A: Next.js 새 프로젝트](./BLOG_OPTION_A_NEXTJS.md)
> - [옵션 B: Vite 프로젝트 확장](./BLOG_OPTION_B_VITE.md)

---

## 핵심 비교표

| 항목 | 옵션 A (Next.js) | 옵션 B (Vite) |
|-----|-----------------|---------------|
| **접근 방식** | 새 프로젝트 생성 | 기존 프로젝트 확장 |
| **프레임워크** | Next.js 14 (App Router) | Vite + React |
| **렌더링** | SSR/SSG/ISR | CSR (클라이언트) |
| **CMS** | Sanity (헤드리스) | 로컬 마크다운/JSON |
| **예상 시간** | 13-20시간 | 9-11시간 |
| **난이도** | 중상 | 중 |
| **SEO** | 완벽 지원 | 제한적 (prerender 필요) |
| **확장성** | 높음 | 중간 |
| **유지비용** | 낮음 (일부 제한) | 무료 |

## 상세 비교

### 1. SEO & 소셜 공유

| 기능 | 옵션 A | 옵션 B |
|-----|-------|-------|
| 검색 엔진 인덱싱 | 100% | 부분적 (prerender 필요) |
| 카카오톡 링크 미리보기 | 완벽 | prerender 없이 불가 |
| 페이스북 OG 이미지 | 완벽 | prerender 없이 불가 |
| 동적 메타태그 | 서버에서 생성 | 빌드 시 고정 |
| sitemap.xml | 자동 생성 | 수동 또는 플러그인 |
| RSS 피드 | API로 제공 | 정적 파일 생성 필요 |

**결론**: SEO가 중요하면 → **옵션 A**

---

### 2. 콘텐츠 관리

| 기능 | 옵션 A | 옵션 B |
|-----|-------|-------|
| 콘텐츠 에디터 | Sanity Studio (GUI) | 마크다운 직접 편집 |
| 비개발자 접근성 | 높음 | 낮음 (Git 필요) |
| 이미지 관리 | Sanity CDN 자동 최적화 | 수동 (public 폴더) |
| 버전 관리 | Sanity 히스토리 | Git 히스토리 |
| 미리보기 | 드래프트 미리보기 가능 | 로컬 개발 서버에서만 |

**결론**: 비개발자가 글 작성하면 → **옵션 A**

---

### 3. 성능

| 지표 | 옵션 A | 옵션 B |
|-----|-------|-------|
| 초기 로딩 (FCP) | 빠름 (HTML 즉시) | 느림 (JS 다운로드 후) |
| 상호작용 (TTI) | 빠름 | 중간 |
| 빌드 속도 | 중간 | 매우 빠름 |
| 번들 크기 | 작음 (코드 스플리팅) | 중간 |
| 캐싱 | ISR (자동 갱신) | 정적 (재배포 필요) |

**결론**: 초기 로딩 속도 중요하면 → **옵션 A**

---

### 4. 개발 경험

| 항목 | 옵션 A | 옵션 B |
|-----|-------|-------|
| 학습 곡선 | 높음 (App Router, Sanity) | 낮음 (기존 스택 유지) |
| 초기 설정 | 복잡함 (환경 변수 많음) | 간단함 |
| 디버깅 | 서버/클라이언트 구분 필요 | 클라이언트만 |
| 타입스크립트 지원 | 완벽 | 완벽 |
| Hot Reload | 좋음 | 매우 좋음 (Vite) |

**결론**: 빠르게 시작하려면 → **옵션 B**

---

| 기능 | 옵션 A | 옵션 B |

| 관련 포스트 | 동일 | 동일 |
| 카테고리 필터 | 동일 | 동일 |

**결론**: 조회수 추적 필요하면 → **옵션 A**

## 추천 시나리오

### 옵션 A를 선택해야 할 때

1. **SEO가 비즈니스에 중요한 경우**
   - 검색 유입이 주요 트래픽 소스
   - 콘텐츠 마케팅 전략 실행

2. **비개발자가 콘텐츠 관리해야 할 때**
   - 마케팅팀이 직접 블로그 작성
   - 편집자/라이터가 별도로 있음

3. **장기적 확장 계획이 있을 때**
   - 다국어 지원 예정
   - 뉴스레터, 회원 시스템 추가 예정
   - 트래픽 증가 예상

4. **소셜 마케팅을 적극 활용할 때**
   - 카카오톡, 페이스북 공유가 중요
   - 링크 미리보기가 필수

---

### 옵션 B를 선택해야 할 때

1. **빠른 MVP가 필요한 경우**
   - 2주 내 런칭 목표
   - 블로그 기능 테스트 단계

2. **개발자가 직접 콘텐츠 관리할 때**
   - 팀 내 개발자가 포스트 작성
   - Git 워크플로우에 익숙함

3. **예산과 리소스가 제한적일 때**
   - 외부 서비스 최소화
   - 유지보수 부담 줄이기

4. **SEO가 덜 중요한 경우**
   - 기존 고객 대상 콘텐츠
   - 내부 지식 베이스 용도
   - 소셜/이메일로 직접 배포

## 마이그레이션 전략

### 옵션 B → 옵션 A 전환 (나중에)

옵션 B로 시작해도 나중에 옵션 A로 마이그레이션 가능합니다.

**재사용 가능 (80%):**
- 모든 React 컴포넌트
- Tailwind 스타일
- 유틸리티 함수
- 타입 정의
- Giscus 댓글 설정

**변경 필요 (20%):**
- 라우팅 (`react-router` → `next/navigation`)
- 데이터 페칭 (로컬 → Sanity)
- 이미지 (`<img>` → `next/image`)
- 링크 (`<Link to>` → `<Link href>`)

## 최종 결정 체크리스트

### 옵션 A 선택 조건 (3개 이상 해당 시)

- [ ] 검색 엔진 유입이 중요하다
- [ ] 비개발자가 콘텐츠를 작성한다
- [ ] 카카오톡/페이스북 공유가 자주 된다
- [ ] 조회수/인기글 기능이 필요하다
- [ ] 장기적으로 기능 확장 예정이다
- [ ] Next.js 학습에 시간 투자할 의향이 있다

### 옵션 B 선택 조건 (3개 이상 해당 시)

- [ ] 2주 내 빠르게 런칭해야 한다
- [ ] 개발자가 직접 글을 작성한다
- [ ] 기존 Vite 프로젝트를 유지하고 싶다
- [ ] 외부 서비스 의존성을 줄이고 싶다
- [ ] SEO보다 기존 채널 배포가 주력이다
- [ ] 추후 마이그레이션 가능성을 열어두고 싶다
