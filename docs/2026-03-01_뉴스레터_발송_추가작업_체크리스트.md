### Phase 1: 이메일 발송 인프라 구축 (필수, 우선순위: 높음)

#### 1.1 이메일 발송 서비스 선택 및 설정
- [ ] 이메일 서비스 선택 (아래 옵션 중 하나)
  - [ ] **Resend** (추천) - 무료: 월 3,000통, 쉬운 설정, 좋은 도달률
  - [ ] **SendGrid** - 무료: 일 100통 (월 3,000통), 복잡한 설정
  - [ ] **Amazon SES** - 저렴, 복잡한 설정, AWS 계정 필요
  - [ ] **Mailgun** - 무료: 월 1,000통, 중간 난이도

- [ ] 선택한 서비스 가입 및 계정 설정
- [ ] API 키 발급
- [ ] 도메인 인증 (SPF, DKIM, DMARC)
  - [ ] DNS 레코드 추가
  - [ ] 발신 도메인: `hello@bujatime.com` 또는 `newsletter@bujatime.com`
  - [ ] 인증 완료 확인 (스팸 방지)

#### 1.2 환경 변수 설정
- [ ] `.env.local`에 API 키 추가
  ```bash
  # 예시 (Resend)
  RESEND_API_KEY=re_xxxxxxxxxxxxx

  # 발신자 정보
  NEWSLETTER_FROM_EMAIL=hello@bujatime.com
  NEWSLETTER_FROM_NAME=부자타임
  ```
- [ ] Supabase Edge Function 환경 변수 설정
  ```bash
  supabase secrets set RESEND_API_KEY=re_xxxxxxxxxxxxx
  ```

## 🛠 기술 스택 추천

### 이메일 발송 서비스 비교

| 서비스 | 무료 한도 | 장점 | 단점 | 추천도 |
|--------|-----------|------|------|--------|
| **Resend** | 월 3,000통 | 쉬운 설정, React 템플릿 지원, 좋은 도달률 | 상대적으로 신규 | ⭐⭐⭐⭐⭐ |
| SendGrid | 일 100통 | 많은 기능, 안정적 | 복잡한 설정, 무료 한도 적음 | ⭐⭐⭐ |
| Amazon SES | 월 62,000통 (EC2 사용 시) | 매우 저렴 | AWS 설정 복잡, 처음엔 샌드박스 | ⭐⭐⭐ |
| Mailgun | 월 1,000통 | 중간 난이도 | 무료 한도 적음 | ⭐⭐ |

**결론: Resend 추천** (간단하고 무료 한도 넉넉)

1. **스팸 방지**

   - 구독 취소 링크 필수 포함
